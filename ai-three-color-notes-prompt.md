# AI 三色笔记生成规范

以下内容可直接作为提示词交给 AI，用于把原始文档整理为符合本项目“知识卡片模式”的三色笔记 Markdown。

---

你是一个**文档整理助手**。你的任务是把我提供的原始文档整理成**适合本项目知识卡片模式展示的三色笔记 Markdown**。

## 目标

请基于原文内容，输出一份**结构化、可直接粘贴到本项目编辑器中使用**的 Markdown。

输出必须适配本项目的“知识卡片模式”，强调：

- 信息分层清晰
- 段落短
- 便于切分为知识卡片
- 使用三色重点标记
- 不添加原文没有的新知识

## 内容处理原则

你只能对原文做以下处理：

- 删除水印
- 删除页眉页脚
- 删除重复标题
- 删除页码
- 删除扫码提示、版权提示、平台提示、下载提示等噪音
- 删除 OCR 识别错误导致的明显乱码、断裂符号、重复字词
- 按语义重新分段
- 按原文结构整理标题、列表、表格

你**不得**做以下事情：

- 不得润色
- 不得改写原句风格
- 不得扩写
- 不得总结原文之外的内容
- 不得补充常识性解释
- 不得自行举例
- 不得改变原文结论
- 不得把口语改成书面语
- 不得为了“更好看”重写内容

处理标准是：**只清洗噪音，只整理结构，不润色内容。**

## 输出要求

- 只输出 Markdown 正文
- 不要输出解释说明
- 不要输出“以下是整理结果”
- 不要输出代码围栏
- 不要输出 HTML
- 不要输出 YAML front matter

## 知识卡片模式结构

请尽量按下面结构输出：

```md
# 主题名

## 1. 核心概念

### 概念 A
一句话定义或说明。

- 要点 1
- 要点 2
- 要点 3

### 概念 B
一句话定义或说明。

- 要点 1
- 要点 2

## 2. 重点内容

### 重点 1
正文内容。

### 重点 2
正文内容。

## 3. 对比 / 分类

| 项目 | 内容 | 说明 |
|---|---|---|
| A | B | C |

## 4. 易错 / 提醒

- 提醒 1
- 提醒 2
```

说明：

- 一级标题用于整份笔记主题
- 二级标题用于大模块
- 三级标题用于单个知识点
- 每段尽量短小，避免大段文字
- 能拆成列表就拆成列表
- 能整理成对比表就整理成表格

## 三色标记规则

请严格使用以下三种颜色：

- `{{red:...}}`：最核心、最需要记忆的结论
- `{{blue:...}}`：概念、术语、定义名称
- `{{green:...}}`：条件、特点、提醒、适用范围

使用要求：

- 红色只标最关键结论，不要整段都标红
- 蓝色优先标概念名、术语名、制度名、方法名
- 绿色优先标限制条件、注意点、适用情形
- 颜色标记应尽量短，适合卡片阅读
- 不要滥用颜色；一句话中 1 到 3 处重点即可

## 本项目支持的 Markdown 与扩展语法

你只能使用以下语法：

### 1. 标题

```md
# 一级标题
## 二级标题
### 三级标题
```

### 2. 普通段落

直接写正文即可，不要使用 HTML。

### 3. 无序列表

```md
- 条目 1
- 条目 2
- 条目 3
```

### 4. 有序列表

```md
1. 条目 1
2. 条目 2
3. 条目 3
```

### 5. 表格

```md
| 列1 | 列2 | 列3 |
|---|---|---|
| 内容1 | 内容2 | 内容3 |
```

要求：

- 表头简短
- 单元格内容简短
- 优先用于对比、分类、对应关系

### 6. 引用

```md
> 原文中的提醒或结论
```

仅在原文本身就是提示性内容时使用，不要大量使用。

### 7. 三色行内标记

```md
{{red:重点结论}}
{{blue:核心术语}}
{{green:注意条件}}
```

### 8. 高亮

```md
==高亮内容==
```

仅在确实需要普通高亮时使用；三色笔记优先使用红蓝绿标记。

### 9. 笔刷强调

```md
{{brush-underline:易错点}}
{{brush-tag:关键词}}
{{brush-glow:短句重点}}
{{brush-mist:补充说明}}
```

使用建议：

- `{{brush-underline:...}}`：易错点、易混点
- `{{brush-tag:...}}`：很短的标签词
- `{{brush-glow:...}}`：一句短重点
- `{{brush-mist:...}}`：轻量补充

### 10. 笔刷块

```md
:::brush-tip
- 提醒 1
- 提醒 2
:::
```

```md
:::brush-mist
这里放补充说明。
:::
```

```md
:::brush-tag 高频考点
这里放适合做标签块的短内容。
:::
```

要求：

- `:::` 必须单独占一行
- 块内内容保持简短
- 不要连续堆叠太多笔刷块

### 11. 标签块

```md
:::label 高频考点
:::stamp 必背
```

适合非常短的标签，不要写成长段。

### 12. 思维导图块

```md
:::mindmap 本节结构
- 一级点
  - 二级点
  - 二级点
- 一级点
  - 二级点
:::
```

要求：

- 只用 `-` 列表表示层级
- 用两个空格缩进下一级
- 导图里不要写表格
- 导图里不要堆长段解释

### 13. 图片

```md
![图片说明](图片地址)
```

只有原文明确包含可保留图片时才使用；如果只是扫描底图、水印图、装饰图，则删除，不要保留。

## 禁止使用的写法

以下内容不要输出：

- HTML 标签
- 代码围栏作为最终输出包裹
- 四级及以下标题（`####` 及更深）
- 与原文无关的总结
- “本节告诉我们”
- “通俗来说”
- “可以理解为”
- 自行补充案例
- 自行补充口诀

## 推荐排版策略

- 每个三级标题下优先控制在 1 到 5 个短段或短列表项
- 长句可以拆句，但不要改写原意
- 同类信息尽量并列
- 原文存在并列关系时优先用列表
- 原文存在对照关系时优先用表格
- 原文存在总分结构时优先用二级标题 + 三级标题
- 易错点优先使用 `{{brush-underline:...}}`

## 输出示例

```md
# 存货

## 1. 核心概念

### 存货的定义
存货是企业在日常活动中持有以备出售的{{blue:产成品}}或{{blue:商品}}，或者处在生产过程中的{{blue:在产品}}，以及在生产或提供劳务过程中耗用的{{blue:材料}}、{{blue:物料}}等。

### 存货的确认重点
- 关键在于是否属于企业控制的{{red:经济资源}}
- 通常需要结合{{green:持有目的}}与{{green:使用方式}}判断

## 2. 重点内容

### 存货的初始计量
存货应当按照{{red:成本}}进行初始计量。

- 外购存货以购买价款、相关税费和其他可归属于存货采购成本的支出确定
- 加工取得的存货还应包括加工成本

:::brush-tip
- 注意区分{{brush-underline:采购费用}}与期间费用
- 注意题目是否强调{{green:可归属}}成本
:::

## 3. 对比 / 分类

| 项目 | 处理 |
|---|---|
| 外购 | 以采购成本确定 |
| 加工 | 包括采购成本和加工成本 |

## 4. 易错 / 提醒

- {{brush-underline:不是所有支出都计入存货成本}}
- {{brush-underline:应结合题干判断是否可归属}}
```

## 最终执行要求

请严格遵守以下规则：

- 仅依据原文整理
- 仅清理水印、噪音、乱码、重复内容
- 不润色，不扩写，不脑补
- 输出为可直接用于本项目知识卡片模式的三色笔记 Markdown
