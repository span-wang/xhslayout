# AI 多模式 Markdown 提示词模板

以下内容用于把原始资料整理成可直接粘贴到本项目编辑器的 Markdown。每个模式都包含最适配的结构、关键词规则和可直接复制的提示词模板。

使用时先选择目标模式，再把对应模板交给 AI，并替换模板末尾的原始资料占位内容。

---

## 通用输出规则

你是一个文档整理助手。请基于我提供的原始资料，输出一份可直接用于本项目编辑器的 Markdown。

必须遵守：

- 只输出 Markdown 正文
- 不输出解释说明
- 不输出代码围栏包裹最终结果
- 不输出 HTML
- 不输出 YAML front matter
- 不编造原文没有的信息
- 不改变原文结论
- 可以清理水印、页眉页脚、页码、扫码提示、版权提示、乱码、重复标题
- 可以按语义重新分段、补齐清晰标题、整理列表和表格

## 本项目支持的语法

AI 整理内容时只能使用以下语法。除非我明确要求，不要使用其他 Markdown 方言或 HTML。

### 1. 标题

```md
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
```

使用建议：

- `#` 用于整份内容主题，只出现一次
- `##` 用于章节、题型、文章大段落
- `###` 用于知识点、小节、单题结构
- `####` 只在精读讲义或长文中作为细分小标题，知识卡片和试卷尽量少用

### 2. 段落和换行

```md
普通正文段落。

另一个普通正文段落。
```

使用建议：

- 短段优先
- 不要用 HTML 的 `<p>`、`<br>` 包正文

### 3. 加粗、斜体、下划线、删除线、行内代码

```md
**加粗**
*斜体*
++下划线++
~~删除线~~
`行内代码`
```

使用建议：

- 学习笔记优先用三色标记和笔刷，少量使用加粗即可
- 代码、命令、字段名可用行内代码

### 4. 无序列表

```md
- 条目 1
- 条目 2
  - 子条目 2.1
  - 子条目 2.2
```

### 5. 有序列表

```md
1. 步骤 1
2. 步骤 2
3. 步骤 3
```

使用建议：

- 步骤、流程、题目序号使用有序列表
- 并列要点、特点、分类使用无序列表

### 6. 表格

```md
| 项目 | 内容 | 提醒 |
|---|---|---|
| A | B | C |
```

可选对齐：

```md
| 左对齐 | 居中 | 右对齐 |
|:---|:---:|---:|
| A | B | C |
```

合并单元格：

```md
| 分类 | 项目 | 内容 |
|---|---|---|
| {{table-span:2x1}}资产 | 流动资产 | 现金、存货 |
| {{table-skip}} | 非流动资产 | 固定资产 |
```

使用建议：

- 表格只放短内容
- 对比、分类、条件对应关系优先用表格
- 大段解释不要塞进单元格

### 7. 引用

```md
> 原文中的重要提示、定义或结论。
```

使用建议：

- 只用于原文中特别适合作为提示或引用的句子
- 不要把普通正文大量写成引用

### 8. 代码块

````md
```js
const value = 1;
```
````

使用建议：

- 只有原文包含代码、命令、配置时才使用

### 9. 图片

```md
![图片说明](图片地址)
![图片说明](图片地址 "图片标题")
```

使用建议：

- 只有原文明确需要保留图片时才使用
- 扫描底图、水印图、装饰图不要保留

### 10. 链接

```md
[链接文字](https://example.com)
```

使用建议：

- 只保留原文已有或用户明确要求的链接

### 11. 三色重点

```md
{{red:核心结论}}
{{blue:概念术语}}
{{green:条件提醒}}
{{gold:最终结论}}
```

使用建议：

- `{{red:...}}` 标最重要、必须记住的结论
- `{{blue:...}}` 标术语、概念、制度名、方法名
- `{{green:...}}` 标条件、范围、提醒、例外
- `{{gold:...}}` 标总结性结论或收束句
- 标记内容要短，不要整段套色

### 12. 高亮

```md
==普通高亮内容==
```

使用建议：

- 用于少量普通强调
- 三色重点优先级高于普通高亮

### 13. 笔刷行内强调

```md
{{brush-glow:短句重点}}
{{brush-mist:补充说明}}
{{brush-underline:易错点}}
{{brush-tag:关键词}}
```

使用建议：

- `brush-glow` 用于醒目的短结论
- `brush-mist` 用于轻量补充
- `brush-underline` 用于易错、易混、陷阱
- `brush-tag` 用于很短的标签词

### 14. 笔刷块

```md
:::brush-tip
- 提醒 1
- 提醒 2
:::
```

```md
:::brush-mist
这里放补充说明。
:::
```

```md
:::brush-tag 高频考点
这里放很短的重点内容。
:::
```

支持：

- `:::brush-tip`
- `:::brush-mist`
- `:::brush-glow`
- `:::brush-underline`
- `:::brush-tag 标题`

使用建议：

- `:::` 必须单独占一行
- 块内内容保持简短
- 不要连续堆叠太多笔刷块

### 15. 标签块和印章

```md
:::label 高频考点
:::stamp 必背
```

也可以写成：

```md
:::label 高频考点
:::

:::stamp 必背
:::
```

使用建议：

- 用于很短的标签
- 不要写成长段

### 16. 提示块

```md
:::note 复盘
- 先看概念边界
- 再看题目条件
:::
```

支持任意英文类型名，例如：

```md
:::tip 提醒
这里写提示内容。
:::
```

使用建议：

- 标题写在首行类型名后面
- 块内可以放段落、列表、表格

### 17. 思维导图块

```md
:::mindmap 本节结构
- 一级点
  - 二级点
  - 二级点
- 一级点
  - 二级点
:::
```

使用建议：

- 只用列表表达层级
- 下一级用两个空格缩进
- 导图内不要放表格和长段解释

### 18. 公式和化学式

行内公式：

```md
$E=mc^2$
```

块级公式：

```md
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

化学式：

```md
{{chem:H2O}}
{{chem:CO2 + C -> 2CO}}
```

使用建议：

- 理科、财务公式可用公式语法
- 普通文字里的简单比例或数字不要强行写成公式

### 19. 手动分页

```md
---
```

使用建议：

- 试卷模式需要强制换页时使用
- 普通知识卡片、文章模式尽量少用

## 模板 1：知识卡片模式

适合：章节知识点、考点提炼、定义公式、易错提醒、高密度复习卡。

最佳结构：

```md
# 主题名

:::stamp 必背

:::mindmap 本节结构
- 核心概念
  - 概念 A
  - 概念 B
- 重点规则
  - 规则 A
  - 规则 B
- 易错提醒
  - 易错 A
:::

## 1. 核心概念

### 概念 A
定义：{{blue:概念 A}}是……

- 特点：……
- 条件：{{green:……}}
- 结论：{{red:……}}

### 概念 B
一句话：……

## 2. 重点规则

### 规则 A
核心：{{red:……}}

- 适用：{{green:……}}
- 不适用：{{brush-underline:……}}

:::brush-tip
- 提醒 1
- 提醒 2
:::

## 3. 对比 / 分类

| 项目 | 含义 | 适用 | 易错 |
|---|---|---|---|
| A | …… | …… | …… |
| B | …… | …… | …… |

## 4. 易错 / 速记

- {{brush-underline:易错点 1}}
- {{brush-underline:易错点 2}}
- {{gold:最终结论}}
```

可直接复制的提示词：

```md
你是一个文档整理助手。请把我提供的原始资料整理成适合本项目【知识卡片模式】的 Markdown。

目标：
- 适合高密度复习和小红书知识卡片展示
- 用 `##` 切分大卡片，用 `###` 切分卡片内知识点
- 每个 `###` 下控制在 1 到 5 个短段或列表项
- 优先使用列表和表格，不要输出大段长文

- 必须包含适量三色重点和易错笔刷

结构要求：
- `#` 写总主题
- 开头可放 `:::stamp 必背` 或 `:::label 高频考点`
- 内容复杂时，在开头放 `:::mindmap 本节结构`
- `## 1. 核心概念`
- `## 2. 重点规则`
- `## 3. 对比 / 分类`
- `## 4. 易错 / 速记`

- 用 `定义：`、`一句话：`、`核心：`、`结论：`、`注意：`、`公式：` 开头，便于渲染为重点行
- `{{red:...}}` 标核心结论
- `{{blue:...}}` 标概念和术语

- `{{brush-underline:...}}` 标易错点
- `:::brush-tip` 放短提醒

只能使用本项目支持的语法：
- 标题 `#`、`##`、`###`
- 段落、列表、表格、引用、图片、链接
- 三色重点 `{{red:...}}`、`{{blue:...}}`、`{{green:...}}`、`{{gold:...}}`
- 高亮 `==...==`

- 思维导图块 `:::mindmap 标题`
- 公式 `$...$`、`$$...$$`、化学式 `{{chem:...}}`

禁止：
- 不输出 HTML
- 不输出代码围栏包裹最终结果
- 不输出 YAML front matter
- 不编造原文没有的内容
- 不为了好看而改写原意

原始资料：
【在这里粘贴原始资料】
```

## 模板 2：精读讲义模式

适合：长章节讲义、系统课程、教材精读、需要逐层理解的内容。

最佳结构：

```md
# 主题名

导读：本节要解决的问题是……

:::mindmap 学习路线
- 导读
  - 背景
  - 学习目标
- 第一部分
  - 概念
  - 原理
- 第二部分
  - 案例
  - 结论
:::

## 1. 章节标题

### 1.1 小节标题

概念：{{blue:核心概念}}指……

原理：……

#### 细分点

- 要点 1
- 要点 2

例子：原文案例或材料……

注意：{{green:限制条件或例外}}

结论：{{red:本小节核心结论}}

## 2. 章节标题

### 2.1 小节标题

背景：……

逻辑：……

:::note 复盘
- 本节先看……
- 再判断……
:::

## 3. 本章总结

总结：{{gold:最终收束结论}}
```

可直接复制的提示词：

```md
你是一个文档整理助手。请把我提供的原始资料整理成适合本项目【精读讲义模式】的 Markdown。

目标：
- 适合长内容、教材、课程讲义的逐层阅读
- 保留讲义节奏，不要压缩成过短卡片
- 用 `##` 表示章节，用 `###` 表示小节，用 `####` 表示必要的细分点
- 开头第一段使用 `导读：`，帮助渲染为讲义导读
- 用角色关键词让重点块更清楚

结构要求：
- `#` 写总主题
- 第一段写 `导读：……`
- 内容复杂时加入 `:::mindmap 学习路线`
- 每个 `##` 是一个章节
- 每个 `###` 是一个小节
- 必要时使用 `####` 作为细分标题
- 每节末尾尽量有 `结论：……` 或 `总结：……`

关键词触发规则：
- `导读：`、`背景：`、`目标：`、`问题：` 用于开篇和章节引入
- `概念：`、`定义：`、`原理：`、`机制：`、`逻辑：` 用于核心讲解

- `结论：`、`总结：`、`要点：`、`核心：` 用于收束

- 段落、列表、表格、引用、代码块、图片、链接
- 三色重点 `{{red:...}}`、`{{blue:...}}`、`{{green:...}}`、`{{gold:...}}`
- 高亮 `==...==`
- 笔刷行内强调和笔刷块
- 标签块 `:::label`、`:::stamp`
- 提示块 `:::note 标题`
- 思维导图块 `:::mindmap 标题`
- 公式 `$...$`、`$$...$$`、化学式 `{{chem:...}}`

禁止：
- 不输出 HTML
- 不输出代码围栏包裹最终结果
- 不把教材长文硬压成碎片
- 不编造例子
- 不改变原文结论

原始资料：
【在这里粘贴原始资料】
```

## 模板 3：题目模式

适合：单题解析、错题复盘、逐题讲解、答案和解析并排/后置切换。

最佳结构 A：单题或少量题目的解析卡

```md
# 主题名

## 第 1 题

题干：题目正文……

- A. 选项 A
- B. 选项 B
- C. 选项 C
- D. 选项 D

答案：{{red:B}}

解析：……

技巧：……

易错：{{brush-underline:……}}

总结：{{gold:……}}

## 第 2 题

题干：题目正文……

答案：……

解析：……
```

最佳结构 B：一组练习题

```md
# 主题名

## 单选题

1. 题干……
   - A. 选项 A
   - B. 选项 B
   - C. 选项 C
   - D. 选项 D
   答案：{{red:B}}
   解析：……
   易错：{{brush-underline:……}}

2. 题干……
   - A. 选项 A
   - B. 选项 B
   - C. 选项 C
   - D. 选项 D
   答案：{{red:A}}
   解析：……

## 简答题

1. 题干……
   答案：……
   解析：……
```

可直接复制的提示词：

```md

- 不漏题号
- 不把答案混在题干里
- 不自行补充答案或解析；原文没有答案时写 `答案：原文未提供`
```

## 模板 4：试卷模式

适合：整套试卷、打印版练习卷、A3 预览、题目区与答案区分离。

最佳结构：

```md
# 试卷标题

:::stamp 练习卷

## 一、单选题

1. 题干……
   - A. 选项 A
   - B. 选项 B
   - C. 选项 C
   - D. 选项 D

2. 题干……
   - A. 选项 A
   - B. 选项 B
   - C. 选项 C
   - D. 选项 D

## 二、多选题

1. 题干……
   - A. 选项 A
   - B. 选项 B
   - C. 选项 C
   - D. 选项 D

## 三、简答题

1. 题干……

---

## 答案解析

1. 答案：{{red:B}}
   解析：……

2. 答案：{{red:AC}}
   解析：……
   易错：{{brush-underline:……}}
```

可直接复制的提示词：

```md
你是一个试卷整理助手。请把我提供的原始资料整理成适合本项目【试卷模式】的 Markdown。

目标：
- 适合整套试卷打印和 A3 预览
- 题目区清爽，答案区后置
- `##` 必须用于题型标题，例如 `## 一、单选题`
- 每道题必须用清晰题号开头，例如 `1. 题干……`
- 答案解析集中放在最后的 `## 答案解析`

结构要求：
- `#` 写试卷标题

- 答案区按题号逐题写 `答案：`、`解析：`、`易错：`

关键词触发规则：
```

## 模式选择建议

| 原始资料类型 | 推荐模式 | 结构重点 |
|---|---|---|
| 教材章节、考点汇总 | 知识卡片模式 | `##` 大卡片，`###` 知识点，表格和易错提醒 |
| 长讲义、系统课程 | 精读讲义模式 | 导读、章节、小节、结论 |
| 单题解析、错题本 | 题目模式 | 题干、答案、解析、易错、总结 |
| 整套练习卷、打印卷 | 试卷模式 | 题型分区、题号清晰、答案后置 |
| 教程、观点长文 | 文章模式 | 导语、背景、观点、案例、结论 |

## 最小检查清单

交给 AI 前，先确认：

- 是否选择了目标模式
- 是否要求只输出 Markdown 正文
- 是否包含“本项目支持的语法”
- 是否禁止 HTML 和代码围栏包裹最终结果
- 是否说明不能编造原文没有的信息

拿到 AI 输出后，快速检查：

- 是否只有一个 `#` 总标题
- `##` 是否承担了模式中最重要的分区
- 题目和试卷是否保留了清晰题号
- 是否使用了正确的角色关键词
- 是否没有滥用颜色、标签和提示块
